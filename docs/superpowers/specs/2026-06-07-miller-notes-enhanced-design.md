# Miller's Anesthesia Notion 筆記強化設計

**日期：** 2026-06-07  
**目標：** 強化 Miller's Anesthesia → Notion 筆記 pipeline，使筆記更詳細、保留原文圖表、並整合記憶輔助結構。

---

## 需求摘要

- 內文改為條列式完整句子（非段落散文）
- 圖表重建為 Notion table block
- 每頁底部加三個記憶輔助區塊：臨床重點、易考點、常見陷阱
- 因果說明：原文有寫的就寫出來，原文沒有的不自行補充

---

## Sub-page 結構

每個 Notion sub-page 固定格式如下：

```
> Miller's Anesthesia 10th ed. Ch XX — 章節名稱, pp. XXXX–XXXX（PDF pp. XXXX–XXXX）

## 一、內文筆記
• 完整句子描述一個概念，原文有因果就一起寫出來。
  - 子條目補充細節。
• 下一個概念。

## 二、圖表
（有表格 → 重建為 Notion table block）
（有流程圖/解剖圖 → 條列文字摘要圖的核心訊息）
（無圖表 → 此區塊省略）

## 三、臨床重點
⚡ 直接與臨床操作相關的一句話重點（3–5 條）

## 四、易考點
❓ 問題形式 → 答案（3–5 條）

## 五、常見陷阱
⚠️ 錯誤認知 → 正確觀念（2–3 條）
```

### 內文條列原則

- 每條一個完整句子，一個概念
- 原文有說明機制/原因 → 用「——」或「，因為」連接
- 原文只陳述事實 → 只寫事實，不自行補充機制
- 必要時加縮排子條說明細節，不堆疊成段落

### 圖表處理原則

| 圖表類型 | 處理方式 |
|---------|---------|
| 表格（藥物比較、數值、分類） | 重建為 Notion table block |
| 流程圖、決策樹 | 條列文字摘要核心步驟/判斷點 |
| 解剖圖、示意圖 | 條列文字描述圖的關鍵訊息 |
| 無圖表的頁面 | 省略「二、圖表」區塊 |

流程圖/解剖圖不重建原因：Notion block 無法還原複雜圖形結構，強制重建反而失真；文字摘要比截圖更省 token 且不需 image hosting。

---

## 處理流程

1. PyMuPDF 渲染該頁為 PNG（1.8x zoom）→ 存 `/tmp/`
2. `Read` 工具讀取圖片，Claude 以視覺方式合成內容
3. 輸出上述五區塊格式（無圖表則省略區塊二）
4. `notion-create-pages` 建立 sub-page
5. `notion-fetch` 驗證全文
6. 發現問題 → `notion-update-page` + `replace_content` 修正

**禁止使用** `page.get_text()` 抽取文字（雙欄排版會產生嚴重亂碼）。

---

## 驗證清單（每頁建完後）

- [ ] 第一行是否有章節來源標注
- [ ] 條列句子是否完整（不截斷）
- [ ] 有無 Unicode 形近錯字（貧血/谼血 等）
- [ ] 有無簡體字漏出
- [ ] 圖表是否正確重建（若有）
- [ ] 三個記憶輔助區塊是否存在且格式正確

---

## 既有已知問題（維持原規範）

1. **PyMuPDF context 污染** — 同 session 曾用文字抽取，當次 session 所有筆記須 notion-fetch 驗證；理想做法是全新 session 只用視覺法。
2. **LLM Unicode 字符混淆** — 每頁建完立即驗證，發現錯字用 replace_content 重寫。
3. **已封存頁面無法編輯** — 需在有效父頁面下重新 notion-create-pages。
