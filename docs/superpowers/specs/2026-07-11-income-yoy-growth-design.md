# 統計頁「收入」模式年增長率 — 設計文件

日期：2026-07-11
狀態：待實作

## 背景

「統計」分頁的下拉選單已經有「儲蓄率／收入／支出」三種模式（見 `docs/superpowers/specs/2026-07-11-stats-mode-dropdown-design.md`）。這次要在「收入」模式加上年增長率（YoY growth）計算。

這是跟「深色莫蘭迪色系改版」一起提出的兩個需求裡範圍較小的一個，先單獨做完上線，色系改版另開一份 spec。

## 範圍

**這次要做的**：`renderStatsIncome()` 每一列收入金額下方，加一行顯示跟列表上一筆比較的年增長率。

**不做的**：儲蓄率／支出模式不受影響；不新增後端 API；不處理年份不連續時的「插值」或「跳過」邏輯（直接跟陣列前一筆比）。

## 1. 計算邏輯

沿用 `renderStatsIncome(years, labelEl, listEl)` 既有的 `years.forEach()` 迴圈，改用帶 index 的 `forEach((y, i) => ...)`：

- 年增長率 = `(y.total_income - years[i-1].total_income) / years[i-1].total_income * 100`
- **陣列第一筆（`i===0`，沒有上一筆可比）不計算、不顯示這行小字**
- 比較對象是「陣列裡的前一筆」，不管年份數字是否連續（例如資料是 2022、2024，中間 2023 因零收入被 `/finance/summary/years` 排除，2024 就直接跟 2022 比）——這是刻意選擇，零收入年份在這個 app 的實際資料裡極少發生，不值得為了處理斷年而增加邏輯複雜度

## 2. UI

在 `renderStatsIncome()` 目前每一列的結構（年份+金額同一行）下方，加一行右對齊小字：

```
2025年 [本年]              NT$1,200,000
                          vs 去年 +20.0%
```

樣式：字級比金額小（比照現有 App 次要文字的 `font-size:12px;color:#64748b` 基底），正值綠 `#4ade80`、負值紅 `#f87171`（沿用既有色碼），格式 `vs 去年 +20.0%` / `vs 去年 -12.3%`。

## 3. 邊界情況

| 情況 | 處理方式 |
|---|---|
| 陣列第一筆（無上一筆可比） | 不顯示年增長率那行 |
| 只有 1 個年度的資料 | 該筆是陣列第一筆，同上不顯示 |
| 上一筆收入為 0 | 理論上不會發生，因為 `/finance/summary/years` 已經排除 `total_income===0` 的年份，不需要額外的除以零防呆 |
| 年份不連續（中間有年度被排除） | 直接跟陣列前一筆比，不特別標註「隔了幾年」 |

## 4. 測試計畫

無後端變更，純前端。用 Playwright，沿用既有 3 年測試資料（2024/2025/2026 收入 NT$1,000,000/1,200,000/800,000）：
1. 切到「收入」模式，確認 2024（陣列第一筆）不顯示年增長率小字
2. 確認 2025 顯示「vs 去年 +20.0%」（綠色）
3. 確認 2026 顯示「vs 去年 -33.3%」（紅色）
4. 確認儲蓄率、支出模式畫面不受影響（regression check）
