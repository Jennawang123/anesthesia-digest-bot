# 歷年儲蓄率比較分頁 — 設計文件

日期：2026-07-11
狀態：待實作

## 背景

用戶提供一張參考截圖（其他 app 的「統計分析」頁面），要在個人財務 App（`finance-app/index.html` + `finance-server/`）裡新增類似的「歷年儲蓄率比較」畫面。截圖原始頁面含 5 個子頁籤（收入分析/支出分析/TOP10/資產/儲蓄率）與 6-icon 底部 nav，但這次只做其中「儲蓄率」這一張比較卡片，做成一個新的頂層分頁，其餘子頁籤留待之後有需要再加。

## 範圍

**這次要做的**：新增一個獨立的「統計」頂層分頁，內容是「歷年儲蓄率比較」卡片。

**不做的**：截圖裡其他 4 個子頁籤（收入分析/支出分析/TOP10/資產）、底部 6-icon nav 改版、年份列表的點擊互動。

## 1. 後端 API

新增 `GET /finance/summary/years`（`finance-server/finance_api.py` + `finance_db.py`）：

- 掃描 `income` 與 `transactions` 兩張表出現過的所有年份（`year_month`/`date` 取年份前綴，取聯集），由舊到新排序
- 排除 `total_income == 0` 的年份（視為無資料，不回傳該年）
- 每個年份沿用既有 `yearly_summary(year)` 邏輯計算，回傳陣列：

```json
[
  {"year": "2022", "total_income": 1027677, "total_expense": 470106, "balance": 557571},
  {"year": "2023", "total_income": 1188643, "total_expense": 493136, "balance": 695507}
]
```

- 沿用既有 `_auth(x_api_key)` header 驗證
- 不新增資料表或欄位

## 2. 前端 UI

**分頁進入點**：頂部 tab bar（目前 `支出/收入/投資/資產`）加第 5 個 `<button class="tab" onclick="go('stats')">📊 統計</button>`，對應 `<div id="page-stats" class="page">`。`go()` 的 switch 加 `stats` case，切換到此頁時才呼叫 `loadStats()`（沿用其他 tab「切換時才 fetch」的 pattern，例如 `income` tab 切換時呼叫 `loadIncome()`/`loadYearlyBalance()`）。

**卡片內容**（一張 `.card`，由舊到新逐年列出）：

- 每列：年份文字（今年加「本年」圓角 outline 標籤）＋右側儲蓄率百分比（正值綠 `#4ade80`／負值紅 `#f87171`，沿用既有色碼）
- 綠色進度條：寬度以 `rate` 相對 `0%~100%` 為滿版基準；`rate > 100` 時寬度截斷在 100%（文字仍顯示實際數字）；`rate < 0` 時進度條寬度為 0（不畫負向條）
- 條下方一行：左「收入 NT$xxx」／右「支出 NT$xxx」，千分位用 `toLocaleString()`，字級比照現有次要文字樣式
- 列表最末：分隔線 + 「歷年平均」＋ 各年儲蓄率（含本年）的算術平均

**樣式**：完全沿用現有卡片/顏色系統（`.card`/`.label`、既有綠紅色碼），不另外設計新樣式語彙。

## 3. 計算邏輯

儲蓄率公式沿用現有「年度結餘」邏輯：`(total_income - total_expense) / total_income * 100`，即全部收入來源、全部支出類別（不像截圖公式排除業外收入或特定支出類別，因為現有 `income.source` 是自由文字、無固定分類）。

歷年平均 = 列表中每個年份儲蓄率的算術平均（非加權），本年（即使資料只到年中）一併納入計算，跟截圖公式一致。

## 4. 邊界情況

| 情況 | 處理方式 |
|---|---|
| 某年 `total_income == 0` | 後端已排除，前端不會拿到該年，不顯示列 |
| 儲蓄率 > 100% | 進度條寬度截斷在 100%，文字顯示實際數字（可能 >100%） |
| 儲蓄率 < 0%（支出>收入） | 文字紅色，進度條寬度為 0 |
| 今年（未跑完整年） | 加「本年」標籤，正常計入列表與平均 |
| 年份列表互動 | 純顯示，不可點擊（先做最小版本） |

## 5. 測試計畫
