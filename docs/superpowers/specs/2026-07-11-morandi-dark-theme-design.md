# 深色莫蘭迪色系改版 — 設計文件

日期：2026-07-11
狀態：待實作

## 背景

個人財務 App（`finance-app/index.html`）目前是深色主題但用鮮豔色（藍黑背景 `#0f172a`/`#1e293b`，語意色綠 `#4ade80`、紅 `#f87171`、藍 `#38bdf8`）。這次要整體改成暖色調、低飽和度的「莫蘭迪」深色系。

這是跟「收入年增長率」一起提出的兩個需求裡範圍較大的一個，已拆成獨立 spec（年增長率見 `docs/superpowers/specs/2026-07-11-income-yoy-growth-design.md`，已上線）。

## 範圍

**這次要做的**：
- 核心色（背景、卡片背景、主/次/輔助文字、綠/紅語意色、強調藍、邊框）改成 CSS 變數 + 莫蘭迪色值，涵蓋全部 5 個分頁（支出/收入/投資/資產/統計）
- 支出類別圓餅圖的十色 `CAT_COLORS` 改成莫蘭迪配色（已用 dataviz skill 的驗證器跑過六項檢查）

**這次不做的**：約 20 個只在特定小地方用 1-4 次的零星色（淨資產卡片漲跌 pill 的部分色階、期貨損益顏色、持股市場標籤色等）維持現狀，之後有需要再單獨開一輪處理。不改版面配置、不改字級/間距，純換色。

| `#f87171` | `var(--red)` |

## 3. 配色驗證（dataviz skill）

用 `scripts/validate_palette.js` 對支出類別十色跑過六項類別色檢查（`--mode dark --surface "#2A2622" --pairs all`，因為圓餅圖任兩色塊都可能相鄰，用全配對檢查而非只檢查相鄰配對）：

- ✅ 亮度區間（OKLCH L 0.48–0.67）全數通過
- ✅ 飽和度下限（OKLCH C ≥ 0.10）全數通過
- ✅ 對比度（跟 `--bg-card` 比 ≥ 3:1）全數通過
- ⚠️ 色盲安全（CVD ΔE）：9 對通過建議值 12，僅「贈與」`#A4921E` 跟「日常」`#D4713A` 這一對在色盲情境下 ΔE 7.1，略低於建議下限 8。**這是刻意接受的例外**：App 的圓餅圖旁邊本來就會顯示文字 legend（類別名稱＋金額＋百分比，見既有 `renderCategoryChart()` 邏輯），顏色從來不是唯一的辨識依據，符合 dataviz skill「identity 不能只靠顏色」的核心原則，所以這個 ΔE 落差可以接受，不需要為了湊到 12 繼續犧牲莫蘭迪的低飽和特性。

背景/文字/語意色的對比度也用同一支驗證器的 `contrast()` function 個別核對過（WCAG 文字對比 ≥4.5:1 一般文字）：`--text-primary` 對 `--bg-card` 11.34:1、`--text-secondary`/`--text-muted` 分別約 5.6:1/4.36:1、`--green`/`--red`/`--blue` 皆 ≥5:1，均通過。

## 4. 邊界情況

| 情況 | 處理方式 |
|---|---|
| 零星的長尾色（約 20 個，1-4 次用量） | 這次不動，維持現有 hex 值，後續視覺一致性若有落差再開下一輪處理 |
| Chart.js 圖表（支出類別圓餅圖、淨資產折線圖） | Chart.js 的顏色設定是 JS 內嵌字串（`backgroundColor:colors` 等），不是 CSS，圓餅圖十色直接改 `CAT_COLORS` 物件值；淨資產折線圖目前用的 `#60a5fa` 屬於長尾色範圍外的既有邏輯，這次一併統一成新的 `--blue`（`#7CAFCE`）對應的 hex 值，因為它在核心色列表裡（9 次用量，屬於核心色） |
| CSS 變數瀏覽器相容性 | 目標平台是 iOS Safari PWA，CSS 自訂屬性（`:root { --x }`）在現代 Safari 全面支援，不需要 fallback |

## 5. 測試計畫

無後端變更，純前端換色。用 Playwright，對 5 個分頁（支出/收入/投資/資產/統計）分別截圖比對改版前後：
1. 確認整體背景從藍黑轉為暖炊灰棕，卡片色階可辨識、跟背景有層次
2. 確認金額正負色（綠/紅）在新背景上清楚可辨，且比改版前柔和
3. 確認支出分頁跟統計分頁「支出」模式的類別圓餅圖十色可以互相區分，legend 文字仍然是主要辨識依據
4. 確認下拉選單、badge、進度條等小元件的邊框/底色套用新變數後沒有跑版或看不清楚
5. 檢查瀏覽器 console 沒有新增錯誤（CSS 變數打字錯誤會讓瀏覽器 fallback 成 `unset`，肉眼比對顏色時容易發現）
